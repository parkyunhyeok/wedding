# 모바일 청첩장 — GitHub Pages/EdgeOne + Google Sheet/Drive(Apps Script)

관리자 모드에서 수정한 내용이 **모든 PC·모바일에 반영**되도록, 백엔드 저장소를
**구글 스프레드시트(텍스트)** 와 **구글 드라이브(사진·썸네일)** 로 바꾼 버전입니다.
정적 사이트에는 서버가 없으므로 **구글 앱스 스크립트 웹앱**이 백엔드 API 역할을 합니다.

```
[방문자 브라우저]  --GET  ?action=get-->  [Apps Script 웹앱]  --읽기-->  [Sheet 텍스트 + Drive 사진]
[관리자 패널]      --POST (비밀번호)-->    [Apps Script 웹앱]  --쓰기-->  [Sheet / Drive]
        ▲                                                                    │
        └──────────────── 정적 호스팅: GitHub → EdgeOne ────────────────────┘
```

- **텍스트**(인사말·이름·일시·장소·교통·계좌) → 스프레드시트 `모바일청첩장_DB`
- **사진/대표사진/썸네일** → 구글 드라이브 폴더(관리자 패널에서 업로드하면 즉시 저장)
- **배경음악** → 저장소의 정적 파일 `assets/bgm.mp3` (EdgeOne CDN이 안정적으로 스트리밍, 교체는 재배포)

## 폴더 구성

```
청첩장 GAS버전/
├─ index.html                # 청첩장 본체(단일 파일, 폰트 임베드). GAS_ENDPOINT 만 채우면 됨
├─ assets/
│  └─ bgm.mp3                # 배경음악(정적)
├─ apps-script/
│  └─ Code.gs                # 백엔드(앱스 스크립트에 붙여넣을 코드)
└─ README.md
```

바인딩 대상(코드에 이미 입력됨):
- 스프레드시트 `모바일청첩장_DB` : `1M2BlLdvKB8FRy9xY6IURZxrwYc657UfqrpG3BRsk8ao`
- 드라이브 폴더(청첩장 썸네일) : `1pClChKU02rh5JKMYY6hL7GuVrUOp5KSP`
- 공유 썸네일(og:image) 파일 : `1GyZu3lYn0Rwsiw2_cKVHznHEOaLWrjfy`

## 1단계 — 앱스 스크립트 백엔드 배포

1. 스프레드시트 `모바일청첩장_DB` 를 연다 → 상단 메뉴 **확장 프로그램 → Apps Script**.
2. 기본 `Code.gs` 내용을 지우고 **`apps-script/Code.gs` 전체를 복사해 붙여넣기** → 저장(💾).
3. (선택) 편집기에서 함수 `setup` 을 한 번 실행 → 권한 승인 팝업 허용
   → 시트에 `정보`·`계좌`·`갤러리` 탭과 기본값이 자동 생성된다.
4. 우측 상단 **배포 → 새 배포** → 톱니바퀴에서 유형 **웹 앱** 선택
   - **실행:** 나(소유자 계정)
   - **액세스 권한:** **모든 사용자**
   → **배포** → 표시되는 **웹 앱 URL**(끝이 `/exec`)을 복사.
5. (선택·권장) **프로젝트 설정 → 스크립트 속성**에 `ADMIN_PASSWORD` 를 등록해
   기본 비밀번호(`1010`)를 원하는 값으로 바꾼다.
6. (선택) 썸네일 교체 기능을 쓰려면 편집기 좌측 **서비스(+)** → **Drive API** 추가.
   추가하지 않아도 나머지 기능은 모두 동작한다.

> ⚠️ 코드를 수정하면 **배포 → 배포 관리 → 편집(연필) → 버전 '새 버전' → 배포**로
> 재배포해야 반영됩니다. `/exec` URL 은 재배포해도 그대로 유지됩니다.

## 2단계 — index.html 에 백엔드 주소 연결

`index.html` 을 열어 상단 설정의 한 줄을 방금 복사한 URL로 바꾼다:

```js
var GAS_ENDPOINT = 'PASTE_APPS_SCRIPT_WEB_APP_URL_HERE';
//                 ↓ 예시처럼 /exec 로 끝나는 실제 URL
var GAS_ENDPOINT = 'https://script.google.com/macros/s/AKfycb....../exec';
```

이 값을 넣기 전에는 서버 동기화 없이 **임베드 기본 텍스트로만** 표시됩니다(관리자 저장 불가).

## 3단계 — 배경음악(선택)

다른 곡으로 바꾸려면 `assets/bgm.mp3` 파일을 교체하고 재배포한다. 파일명을 바꾸려면
`index.html` 의 `var BGM_SRC = 'assets/bgm.mp3';` 도 함께 수정한다.

## 4단계 — GitHub 푸시 + EdgeOne 연동

1. 이 폴더(`청첩장 GAS버전`) 내용을 **GitHub 저장소에 올린다**
   (권장: 이 파일들이 저장소 **루트**에 오도록. `index.html` 이 최상위여야 함).
2. **EdgeOne Pages → 프로젝트 → GitHub 저장소 연동** →
   빌드 없음(정적) / 루트 디렉터리를 `index.html` 이 있는 위치로 지정 → 배포.
3. 이후 `git push` 하면 EdgeOne 이 자동 재배포한다.

> EdgeOne 이 하위 폴더를 루트로 지정하는 걸 지원하지 않으면, `index.html`·`assets/`
> 를 저장소 최상위로 옮겨 주세요. `apps-script/`·`README.md` 는 사이트 동작과 무관합니다.

## 관리자 사용법

1. 페이지 **맨 아래 여백(Thank you 아래)을 5번 탭** → 비밀번호 입력창.
2. 비밀번호 입력(기본 `1010`, 또는 스크립트 속성으로 지정한 값) → 관리자 패널.
3. **텍스트**(인사말·이름·일시·장소·교통·계좌) 수정 → **변경사항 저장** → 시트에 기록
   → 다른 기기는 새로고침 시 반영.
4. **대표사진·갤러리·썸네일**은 파일을 고르면 **업로드 즉시 드라이브에 저장·반영**
   (별도 저장 불필요). 갤러리는 순서 변경·삭제·대표사진 지정 가능.

## 데이터 저장 구조

**스프레드시트 탭**
- `정보` : `key | value` (greeting, groom_name, …, venueAddress, transport_*, coverFileId, dataVersion, updatedAt)
- `계좌` : `side | label | bank | holder | number`
- `갤러리` : `fileId` (행 순서 = 표시 순서)

**드라이브 폴더** : 업로드된 사진 원본이 저장됨. 이미지는
`https://lh3.googleusercontent.com/d/<파일ID>=w1600` 형식으로 표시(업로드 시 자동으로
'링크가 있는 모든 사용자 보기'로 공개 설정).

## 링크 공유하기

맨 아래 **청첩장 링크 공유하기** 버튼은 모바일에서 OS 공유 시트를 띄워
카카오톡·문자·인스타그램 DM 등으로 바로 공유합니다(`navigator.share`).
공유 시트를 지원하지 않는 PC 브라우저에서는 **링크 복사**로 대체됩니다.
카카오톡 등에 뜨는 **미리보기 이미지·문구**는 `index.html` `<head>` 의 `og:image`·`og:title`·`og:description` 을 따릅니다.

## 주의사항 / 한계

- **CORS**: 프런트의 POST 는 커스텀 헤더 없이 문자열 본문만 보내 프리플라이트를
  피합니다. Code.gs 를 고칠 때 응답을 `ContentService` JSON 으로 유지하세요.
- **이미지/드라이브**: 구글 드라이브는 CDN 이 아니라, 하객이 동시에 몰리면 사진 로딩이
  느려질 수 있습니다. 사진 장수를 과하게 늘리지 않는 걸 권장합니다.
- **공유 썸네일 캐시**: 카카오톡·페이스북 등은 미리보기 이미지를 캐시하므로 썸네일을
  바꿔도 즉시 반영되지 않을 수 있습니다(카카오는 캐시 만료 후 갱신).
- **고정 썸네일 교체**: 같은 URL 을 유지하며 내용만 바꾸려면 Apps Script 에 **Drive API
  고급 서비스**가 필요합니다(1단계 6). 없으면 새 파일로 저장되며 `<head>` og:image 는
  수동 교체가 필요합니다.
- **비밀번호**: 정적 HTML 에는 비밀번호가 없습니다(서버에서만 검증). 기본값 `1010` 대신
  스크립트 속성 `ADMIN_PASSWORD` 로 바꾸는 걸 권장합니다.
- `apps-script/Code.gs` 는 소스 참조용입니다. 서비스 동작과 무관하며, 웹 루트에
  노출되는 게 신경 쓰이면 배포 브랜치에서 제외해도 됩니다.

## 배포 후 검증 체크리스트

- [ ] 브라우저에서 `<GAS_ENDPOINT>?action=get` 을 열면 JSON 이 보이는가
- [ ] 관리자 로그인 → 텍스트 저장 → **다른 기기/시크릿 창** 새로고침 시 반영되는가
- [ ] 대표사진/갤러리 업로드 후 사진이 표시되는가(공개 권한 자동 설정)
- [ ] 맨 아래 **공유하기** 버튼이 동작하는가(모바일: 공유 시트 / PC: 링크 복사)
- [ ] 페이지 소스보기에서 비밀번호가 검색되지 않는가(`1010` 검색 → 없어야 함)
- [ ] 빌드 식별자(하단 `build ...`)가 최신인가(캐시된 구버전 판별)
